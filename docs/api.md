# Humanizer-zh-TW 介面定義 (api.md)

本專案作為 AI 技能（Skill），其「API」即為與 AI 代理程式對話時的觸發指令與互動介面。

## 1. 觸發指令 (Trigger Commands)

### `/humanizer-zh`
- **功能**：對輸入的內容進行人性化處理。
- **參數**：`[待處理文本]` 或 `[檔案路徑]`。
- **範例**：
  ```
  /humanizer-zh 請幫我人性化這段文字：[內容]
  ```

## 2. 互動模式 (Interactions)

### A. 文字改寫請求
- **請求**：使用者提供一段具有「AI 味道」的文字。
- **回應**：
    - **人性化版本**：移除 AI 特徵後的自然文本。
    - **修改摘要**（可選）：說明移除哪些模式（如：移除「此外」、「關鍵作用」等詞彙）。
    - **品質評分表**：五個維度的評分及總分。

### B. 檔案處理請求
- **請求**：`/humanizer-zh [FILENAME]`
- **回應**：讀取檔案內容並輸出人性化後的版本。

## 3. 回應格式規範 (Response Schema)

AI 代理程式在執行本技能後，應遵循以下輸出格式：

```markdown
### ✨ 人性化改寫結果
[改寫後的文本]

---
### 🛠️ 修改要點
- 移除 [模式名稱]...
- 增加 [具體細節]...

### 📊 品質評分 (/50)
| 維度 | 得分 |
|------|------|
| 直接性 | /10 |
| 節奏 | /10 |
| 信任度 | /10 |
| 真實性 | /10 |
| 精煉度 | /10 |
| **總分** | **X/50** |
```
